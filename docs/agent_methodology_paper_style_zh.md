# NarrativeKnowledgeWeaver Agent 方法说明

## 1. 问题定义

我们的目标是构建一个面向长篇叙事文本的检索问答系统。与通用开放域问答不同，这类任务通常同时包含以下难点：

- 证据跨越多个段落甚至多个情节阶段
- 局部表述与全局叙事状态之间存在张力
- 许多问题不是单纯事实匹配，而是涉及动机、态度、因果、时序与剧情推进
- 多项选择题往往要求对多个候选答案进行相对比较，而不是仅找到一条支持性证据

因此，我们的核心假设是：叙事问答不能仅依赖单一检索视角，而需要把文本证据、结构化实体关系、叙事层级结构以及交互信息统一到一个可操作的检索与推理框架中。

## 2. 方法总览

整体方法可以概括为一个“先编译、后检索、再决策”的两阶段框架。

第一阶段是离线知识编译。系统将原始文章转换为多种互补的中间表示，包括：

- 文档与句子级文本片段
- 实体与关系图
- 叙事层级结构
- 角色与对象间的 interaction 记录

第二阶段是在线问题求解。面对一个问题，系统不直接生成答案，而是先判断该问题需要何种证据，再调用相应的检索路径，从多视角证据中形成候选结论，最后输出答案。

这一框架的关键思想不是“让一个大模型记住所有文本”，而是“让模型在结构化检索空间中做条件化证据组合”。

## 3. 多层知识表示

### 3.1 局部文本表示

最底层表示是分块后的文本单元，包括文档级块与更细粒度的句子级块。它们承担两项作用：

- 保留原文表述，支撑精确证据定位
- 为稠密与稀疏检索提供基础索引

局部文本表示适合回答显式事实型问题，例如人物、地点、物品属性、出现位置和原文细节。

### 3.2 基础语义图表示

在文本之上，我们进一步抽取实体与关系，形成基础知识图。该图并不追求通用百科式完备性，而是强调叙事文本内部的可检索结构，例如：

- 角色、对象、地点、事件等实体
- 实体之间的行动、关联、属性和上下文关系
- 文档节点与实体节点之间的包含关系

这种图表示的作用主要有两点：

- 提供跨段落的实体归并与关系追踪能力
- 使得问答不再局限于关键词匹配，而能够沿结构进行证据扩展

### 3.3 叙事层级表示

仅有基础实体关系图仍不足以处理剧情推进、角色状态变化和跨章节因果。为此，我们引入显式的叙事层级结构，将文本进一步组织为：

- Event
- Episode
- Storyline

这一层级表示的核心动机是把“局部事件”提升为“中程情节单元”和“长程叙事主线”。这样做的好处是：

- 可以把分散的局部证据归入同一条叙事链条
- 可以在问题涉及“为什么”“如何发展”“暗示了什么”时，提供高层剧情上下文
- 可以缓解长文本中远距离依赖难以直接由局部检索命中的问题

### 3.4 交互表示

许多叙事问题的关键不只是“谁做了什么”，而是“谁与谁如何互动”“某句话的态度是什么”“一次接触改变了什么”。因此，我们额外抽取 interaction 表示，用于记录角色、对象与事件之间的更细粒度交互。

interaction 表示主要服务于：

- 对话与互动检索
- 角色关系变化分析
- 动机与态度类问题的辅助判断

## 4. 检索不是单一路径，而是多视角证据搜索

我们的基本观点是：不同问题天然需要不同证据形态。因此，系统不采用单一检索器，而是把检索建模为“多路径证据搜索”。

### 4.1 局部证据检索

对于精确事实型问题，系统优先从局部文本空间中寻找直接支持证据。此类检索的目标不是抽象总结，而是尽量找到与答案最接近的原文片段。

这种路径特别适合：

- 显式属性
- 明确事件细节
- 文本中可直接引用的短事实

### 4.2 叙事层级检索

对于动机、因果、剧情推进、隐含关系等问题，仅靠局部文本片段往往不够，因为正确答案依赖更高层的叙事语境。此时系统会沿叙事层级进行自上而下或自下而上的搜索，在 storyline、episode 与 event 之间建立上下文约束，再回落到具体文档证据。

这种层级检索的核心作用是把“相关片段”转化为“相关剧情轨迹”。

### 4.3 实体与事件轨迹检索

对于围绕单一角色、对象或关系展开的问题，系统会尝试构建实体或事件轨迹。换言之，系统关注的不是单一命中片段，而是：

- 该实体在文中经历了哪些关键状态变化
- 它与哪些事件相连
- 这些事件如何共同支持最终结论

这类检索特别适用于角色动机、人物状态演变和长期关系判断。

### 4.4 时序解析检索

当问题本质上依赖“先后关系”时，正确答案往往并不取决于单个事实，而取决于多个事实之间的顺序约束。因此系统会显式执行时序解析式检索，将候选证据组织成时间链，以解决：

- before / after
- chronology
- progression
- time comparison

## 5. 检索路由：把问题映射到证据需求

方法的一个核心组成部分是检索路由。这里的目标不是替代理性推理，而是在问题尚未展开求解之前，先估计其证据需求。

我们将问题路由看成一个条件决策过程，其核心输出包括：

- 问题类型
- 答案形态
- 所需证据类型
- 初始检索路径
- 不足时的升级路径

这里最重要的思想是：`no strategy` 不等于无路由。即使系统不读取任何策略记忆，它仍然需要根据问题本身判断应优先查看哪一类证据。否则，agent 会在过大的工具空间中进行低效甚至错误的探索。

我们希望路由器承担的是“限制初始搜索空间”的职责，而不是“直接决定最终答案”。

## 6. 问答被建模为证据聚合，而非单次生成

在在线求解阶段，系统并不把模型看成一次性回答器，而是看成一个在外部证据支持下进行逐步判断的决策器。

一个问题的求解过程通常包含：

1. 判断问题需要什么类型的证据
2. 执行一条或多条检索路径
3. 整理得到的局部、中层与全局证据
4. 对候选答案做比较与校准
5. 输出最终答案

因此，我们的方法与传统“RAG + 直接回答”最大的区别在于：它强调证据形态之间的互补，而不是把所有命中片段简单拼接给模型。

## 7. 多项选择题的特殊处理

多项选择题在叙事问答中尤其困难，因为它要求系统不仅找到支持正确选项的证据，还要区分错误选项为何不成立。

因此，我们不把选择题简单视为普通开放问答，而是单独建模为“候选答案比较”任务。核心思想包括：

- 将每个选项视为一个独立假设
- 分别检索支持和反驳证据
- 比较各个选项的证据强度、直接性与额外假设数量
- 在必要时执行最终的选项适配与校准

这种方式的本质是把选择题从“生成一个答案”改造成“在多个证据支持的假设之间做判别”。

## 8. 策略记忆：从单次成功轨迹中学习检索模式

除了即时检索外，我们还引入策略记忆，用于积累“什么类型的问题通常应采用什么检索路径”。

这里的核心目标不是记住具体答案，而是记住：

- 问题模式
- 适合的证据需求
- 有效的工具链
- 常见失败路径

从方法论上讲，策略记忆相当于在检索层面学习一个任务先验。它试图回答的问题是：

- 这类问题通常属于哪种抽象模式
- 对这种模式而言，最小有效检索链是什么
- 哪些常见尝试虽然看似合理，但通常无效

### 8.1 Pattern 与 Chain 解耦

一个重要设计原则是将问题模式与执行链解耦。

- pattern 负责回答“这是什么任务”
- chain 负责回答“这个任务怎样检索更有效”

这样做的原因是，同一类问题可能存在多个可行检索路径；如果把二者混为一体，策略库就会快速膨胀，并造成大量近重复模板。

### 8.2 最小有效链

我们不希望把一次成功轨迹的所有步骤都写入记忆，因为真实成功轨迹中常包含无效探索和弯路。因此，策略学习更关注最小有效链，即：

- 哪些步骤直接提供最终证据
- 哪些步骤为后续关键步骤提供了必要中间约束
- 哪些步骤没有真正贡献，应被裁掉

这一点对于让记忆可迁移、可合并、可稳定匹配至关重要。

## 9. Offline 与 Online 的互补

我们把策略学习分为 offline 与 online 两种模式。

### 9.1 Offline

offline 阶段通常有更可靠的监督信号，因此适合做：

- 模式抽象
- 最小有效链提取
- 模板合并
- 代表链选择

offline 的目标是得到一个规模较小、模式较干净、泛化性更强的静态策略库。

### 9.2 Online

online 阶段直接服务于运行时，因此必须更保守。它更适合：

- 缓慢积累高价值新模式
- 更新已有模式的支持统计
- 从真实失败中总结反例

从方法上看，offline 更像“系统性蒸馏”，online 更像“增量校正”。

## 10. 为什么这个框架适合叙事问答

我们认为该框架适合叙事问答，主要有三个原因。

第一，叙事问题天然是多粒度的。某些问题只需要局部句子，某些问题却必须依赖整条 storyline。单一检索粒度无法兼顾。

第二，叙事问题天然是高依赖上下文的。人物态度、动机和剧情暗示往往不能由一句话直接读出，而需要结构化上下文进行约束。

第三，选择题天然是比较问题，而不是单纯检索问题。只有把选项显式作为候选假设处理，系统才能更稳定地做区分。

因此，我们的方法并不是单纯追求“更强的生成模型”，而是试图通过多层表示、多路径检索和显式证据比较，把叙事问答转化为一个更可控的结构化决策过程。

## 11. 方法的核心贡献可以概括为

1. 提出了一种面向长篇叙事文本的多层知识编译框架，将局部文本、语义图、叙事层级与 interaction 表示统一到一个可检索空间中。
2. 提出了一种多路径证据搜索机制，使系统能够针对事实、因果、动机、关系和时序等不同题型选择不同证据形态。
3. 将多项选择题显式建模为候选答案比较任务，而不是普通生成任务，从而提升复杂叙事问题下的判别稳定性。
4. 引入 pattern-chain 解耦的策略记忆框架，使系统能够在离线和在线阶段逐步学习更高效的检索路径。

## 12. 一个更简洁的总结

如果用一句话概括整套方法：

我们不是让模型直接“读完整篇文章并回答”，而是先把文章编译成多层叙事知识空间，再让 agent 在这个空间中按需检索、比较和决策。
